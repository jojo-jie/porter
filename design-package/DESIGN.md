# Porter 设计系统

## 视觉主题与氛围

Porter 是一个面向 macOS 的远端文件传输与目录浏览工具。视觉姿态应接近系统工具而非营销产品：温暖、安静、可长时间使用，避免纯白和冷灰带来的生硬感。

核心氛围：

- 暖奶油画布承载主内容，侧栏略微下沉，形成清晰但柔和的工作区。
- 单一暖橙作为操作和路径强调色，只在关键行动、文件夹、聚焦边框和状态徽章中出现。
- 层次主要依赖轻微色差、1px 发丝线和连续圆角，不依赖厚重阴影。
- 文案直接、工具化，强调「主机」「远端目录」「上传」「下载」「路径」等任务语义。

## 色板与角色

浅色模式：

| 角色 | 值 | 用途 |
| --- | --- | --- |
| Canvas | `#F8F7F2` | 主内容背景，带暖调的奶油底色 |
| Surface | `#FFFEFB` | 卡片、输入框、搜索框、状态条 |
| Sidebar | `#F3F1EB` | 左侧主机列表与设置导航背景 |
| Accent | `#CC785C` | 主按钮、目录图标、焦点边框、SSH 徽章 |
| Border | `rgba(0,0,0,0.09)` | 发丝分割线、输入框和卡片边框 |
| Row Highlight | `rgba(0,0,0,0.055)` | 侧栏选中和悬停行 |
| Primary Text | `#24211D` | 标题、主机名、主要路径 |
| Secondary Text | `#68625A` | 说明、状态、元信息 |
| Tertiary Text | `#9B948A` | 标签、提示、表头 |

深色模式：

| 角色 | 值 | 用途 |
| --- | --- | --- |
| Canvas | `#1E1E1E` | 主内容背景，避免纯黑 |
| Surface | `#282828` | 卡片、输入、浮层主体 |
| Sidebar | `#191919` | 左侧区域，使主画布略微前景化 |
| Accent | `#D47D60` | 暖橙强调，在深色中提高可见度 |
| Border | `rgba(255,255,255,0.10)` | 发丝线与控件边框 |
| Row Highlight | `rgba(255,255,255,0.11)` | 侧栏选中和悬停 |
| Primary Text | `#F1ECE5` | 主文本 |
| Secondary Text | `#B8B0A6` | 次级说明 |
| Tertiary Text | `#817A72` | 标签、表头 |

状态色保持系统语义：成功使用柔和绿色，错误使用红色，警告使用橙色；不要让状态色与 Porter 的主暖橙竞争。

## 字体排版规则

优先使用 Apple 系统字体，保持原生 macOS 质感。路径、权限、端口、主机副标题、日志和远端列表元数据使用等宽字体，强化技术工具属性。

| 层级 | 建议规格 | 用途 |
| --- | --- | --- |
| 大标题 | System `34px` / `600` | 当前主机名、关键工作区标题 |
| 页面标题 | System `28px` / `600` | 设置页标题、弹窗主标题 |
| 区块标题 | System `17px` / `600` | 表单组、卡片标题 |
| 主正文 | System `15-16px` / `400-500` | 按钮、主机行、文件名 |
| 次级说明 | System `13-14px` / `400` | 帮助说明、状态文本 |
| 标签与徽章 | System `11-12px` / `500-600`，字距 `0.6-0.8px` | 元信息标签、SSH 徽章、表头 |
| 技术文本 | Monospace `13-15px` / `400` | 路径、权限、日志、端口、大小 |

标题不应过度装饰；层级来自字号、字重和空间，而不是颜色堆叠。长路径和主机名默认允许中间截断或文本选择。

## 布局与构成

主界面采用 macOS 工具型双栏结构：

- 左侧侧栏宽度约 `220-360px`，理想宽度 `280px`，包含搜索、刷新和主机列表。
- 右侧工作区最小宽度约 `760px`，浅色画布上以纵向任务流组织。
- 主内容最大化铺开，但关键表单和设置页可限制在 `720px` 左右，避免横向阅读过宽。
- 工作区横向内边距按窗口宽度调整：紧凑 `18px`，中等 `28px`，常规 `40px`。
- 主要任务流间距使用 `28px`，设置页区块使用 `22px`，卡片内部使用 `20-22px`。

界面结构建议：

1. 侧栏用于选择上下文：主机、设置分区、搜索筛选。
2. 详情顶部展示当前上下文：主机名、SSH 徽章、用户/地址/端口。
3. 中段呈现主要任务：远程路径输入、拖拽上传、主要行动按钮。
4. 底部或浮层 footer 呈现状态：当前路径、上传/下载/重命名结果。

## 组件风格

侧栏主机行：

- 行高紧凑，垂直内边距 `4-5px`，左右内边距 `10px`。
- 主机名使用中等字重，副标题使用等宽 caption，并允许中间截断。
- 选中和悬停使用淡色背景，不使用左侧彩条。

搜索与路径输入：

- 圆角 `9-10px`，背景使用 Surface 的 `85-100%` 不透明度。
- 默认边框为发丝线，聚焦时边框变为 Accent `55%`，线宽可提升到 `1.5px`。
- 路径输入必须使用等宽字体；占位文本应给出真实路径范例，例如 `/var/www/app` 或 `~/uploads`。

按钮：

- 主操作使用系统 prominent 样式并 tint Accent，例如「选择文件上传」「使用此目录」。
- 次操作使用 bordered 或 plain，例如「打开终端」「刷新」「取消」。
- 图标按钮采用 `32px` 圆形点击区，边框发丝线，背景为 Surface `90%`。
- 可点击区域悬停显示手型指针；禁用态保持系统弱化，不额外加彩色。

拖拽上传区：

- 最小高度约 `220px`，圆角 `16px`，内边距 `28px`。
- 默认背景 Surface，边框 Border；拖入时使用 Accent `6%` 背景和 Accent `65%` 边框。
- 图标置于 `64px` 圆形底中，激活时用 Accent `16%` 背景。

远端目录浏览器：

- 浮层尺寸约 `680 x 520px`，圆角 `18px`，阴影 `0 14px 24px rgba(0,0,0,0.22)`。
- 顶部为标题与主机 alias，中段依次是导航面包屑、筛选栏、列表表格。
- 表格列建议：名称、修改时间、大小、类型、操作。名称列自适应，其余列固定宽度。
- 文件夹和面包屑使用 Accent；文件、权限、大小等元数据降低对比度。
- 悬停时才展示下载和重命名操作，保持列表安静。

设置页：

- 与主应用共用双栏导航，不做独立窗口风格。
- 设置卡片圆角 `14px`，内边距 `22px 20px`，背景 Surface `92%`。
- 主题选择器使用胶囊分段控件，选中项由 Surface 背景承载。

## 间距、圆角与交互节奏

基础间距节奏：

- `2-4px`：行内微间距、列表标题与副标题。
- `6-8px`：搜索框内容、侧栏按钮、表头。
- `10-12px`：控件组、按钮组、输入框。
- `14-16px`：状态条、浮层分区、拖拽区内部组。
- `20-28px`：卡片内边距、主工作流区块。
- `40-48px`：页面级留白、设置页左右边距。

圆角节奏：

- `9-10px`：搜索框、输入框、侧栏行。
- `14px`：设置卡片、小信息面板。
- `16px`：拖拽区和大型任务卡片。
- `18px`：浮层容器。
- `999px`：徽章、分段控件、圆形图标按钮。

动效应短促且功能化：

- 聚焦、浮层出现、拖拽态切换使用 `ease-out 150-180ms`。
- 刷新按钮可用轻微 spring 旋转，表达重新读取配置。
- 上传中图标可使用低强度 pulse，避免抢占注意力。

## 层次、边框与效果

Porter 的层次来自「色块关系 + 细边框」，而不是厚重阴影。

- 常规卡片、输入和列表只使用 `1px` 边框。
- 主工作区左侧可使用极轻的横向渐变，模拟 split view 分隔感。
- 只有遮罩浮层和警示型系统弹窗使用阴影。
- 遮罩背景建议为黑色 `18%`，不应完全模糊内容。
- 发丝线永远克制：浅色使用黑色 `9%`，深色使用白色 `10%`。

图标使用 SF Symbols 的单色或层级渲染。文件夹、刷新、上传、终端等图标应保持系统默认线性风格，不引入自定义插画。

## 响应式行为

虽然 Porter 是 macOS 桌面应用，界面仍需适配窄窗口：

- 主机标题与 SSH 徽章在横向不足时从水平排列切到垂直排列。
- 元信息三列在窄宽度下变为纵向堆叠。
- 操作按钮优先水平排列，空间不足时垂直铺满宽度。
- 详情区横向内边距按宽度缩减，确保路径输入和拖拽区仍可操作。
- 远端浏览器表格在极窄视口中可保留横向滚动；不要把所有列挤成不可读文本。

## 建议与禁忌

建议：

- 使用暖色中性色作为背景，保持长时间使用的舒适度。
- 让 Accent 只服务于主行动、路径导航和当前目录语义。
- 对技术文本使用等宽字体，并允许复制和选择。
- 将上传、下载、重命名等结果放进明确状态区域。
- 对危险或远端立即生效操作增加确认文案。

禁忌：

- 不要使用冷蓝作为主色，除非是系统链接或明确外部平台语义。
- 不要堆叠渐变、玻璃拟态或大面积彩色卡片。
- 不要把表格做成营销型卡片网格；远端目录需要可扫读。
- 不要把真实主机名、用户、密钥路径或个人路径写进示例。
- 不要用厚阴影替代边框层次。
